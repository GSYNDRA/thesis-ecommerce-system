\section{Implementation Environment}

\subsection{Technology Stack}

\begin{itemize}
    \item Runtime and API framework: Node.js (ESM) with Express.
    \item Persistence: PostgreSQL with Sequelize ORM.
    \item Coordination and short-lived state: Redis.
    \item Realtime communication: Socket.IO.
    \item Payment integration: MoMo payment API (create payment and IPN callback).
    \item AI integration: provider-based AI service (OpenAI/OpenRouter via config).
\end{itemize}

\subsection{Runtime Initialization}

\begin{itemize}
    \item Server entrypoint initializes PostgreSQL, Redis client, HTTP server, and Socket.IO.
    \item Redis expiry subscribers are started for reservation rollback and staff heartbeat timeout handling.
    \item Environment configuration is validated by Zod schema before startup.
    \item Graceful shutdown closes HTTP server, websocket server, subscribers, Redis, and PostgreSQL connection.
\end{itemize}

\section{Core Feature Implementation}

\subsection{Feature 1: Authentication and Session Security}

Implemented in \texttt{auth.route.js}, \texttt{auth.controller.js}, and \texttt{auth.service.js}.
\begin{itemize}
    \item Registration and verification:
    \texttt{POST /auth/register}, \texttt{GET /auth/verify-email}, \texttt{POST /auth/resend-verification}.
    \item Login and token lifecycle:
    \texttt{POST /auth/login}, \texttt{POST /auth/refresh-token}, \texttt{POST /auth/logout}.
    \item Password recovery:
    \texttt{POST /auth/forgot-password}, \texttt{POST /auth/verify-otp}, \texttt{POST /auth/reset-password}, \texttt{POST /auth/resend-otp}.
    \item Security controls include bcrypt hashing, access/refresh token separation, and session invalidation logic.
\end{itemize}

\subsection{Feature 2: Product Catalog and Query Filtering}

Implemented in \texttt{catalog.route.js} and \texttt{catalog.service.js}.
\begin{itemize}
    \item \texttt{GET /catalog/products}: paginated listing with filter and sorting support.
    \item \texttt{GET /catalog/products/:slug}: detailed product response by slug.
    \item \texttt{GET /catalog/filters}: filter metadata for frontend querying.
    \item Catalog service maps relational product data into API response payloads for list and detail views.
\end{itemize}

\subsection{Feature 3: Cart Lifecycle Management}

Implemented in \texttt{cart.route.js}, \texttt{cart.controller.js}, and \texttt{cart.service.js}.
\begin{itemize}
    \item \texttt{POST /cart/add}: validates variation and stock before append/update.
    \item \texttt{GET /cart}: returns authenticated user's active cart snapshot.
    \item \texttt{PATCH /cart/items/:cartItemId}: quantity update with stock recheck.
    \item \texttt{DELETE /cart/items/:cartItemId}: remove item from active cart.
    \item Service guards include user existence check, active-cart resolution, cart lock protection, and snapshot recomputation.
\end{itemize}

\subsection{Feature 4: Checkout and Reservation Engine}

Implemented in \texttt{checkout.route.js} and \texttt{checkout.service.js}.
\begin{itemize}
    \item \texttt{GET /checkout/preview}: validates cart items, computes subtotal/discount, and checks reservation capacity.
    \item \texttt{POST /checkout/place-order}: creates pending order and payment intent after atomic reservation.
    \item \texttt{GET /checkout/order-status/:orderId}: polls payment and order state.
    \item Redis Lua script performs atomic reservation for stock and voucher counters.
    \item Reservation metadata is cached to support deterministic rollback and post-payment reconciliation.
\end{itemize}

\subsection{Feature 5: Payment Callback and Order Finalization}

Implemented in \texttt{payment.route.js}, \texttt{payment.controller.js}, and \texttt{payment.service.js}.
\begin{itemize}
    \item \texttt{POST /payment/momo/ipn}: receives MoMo callback and verifies signature.
    \item \texttt{GET /payment/momo/redirect}: redirects return params to frontend result route.
    \item Finalization flow updates order state, persists order items/discounts, adjusts stock, and clears purchased cart items.
    \item Callback handling applies idempotent processing strategy for repeated notifications.
\end{itemize}

Implemented across \texttt{chat.route.js}, \texttt{chat.service.js}, \texttt{staff.service.js}, and \texttt{websocket/}.
\begin{itemize}
    \item Session operations:
    \texttt{GET /chat/session/active},
    \texttt{GET /chat/session/:sessionUuid/history},
    \texttt{POST /chat/session/:sessionUuid/request-human},

    \item Staff service handles assignment, heartbeat monitoring, disconnect recovery, and reassignment.
\end{itemize}

\section{Cross-Cutting Engineering Concerns}

\subsection{Validation, Security, and Error Boundaries}

\begin{itemize}
    \item Request validation is enforced at route boundary by Zod schemas and validation middleware.
    \item Protected routes use JWT middleware before service execution.
    \item Security headers are enabled by Helmet and payload compression by \texttt{compression}.
    \item Service-layer errors are propagated to controller-level response handling.
\end{itemize}

\subsection{Observability and Operational Safety}

\begin{itemize}
    \item Winston daily rotation logs are configured for info and error channels.
    \item Reservation expiry subscriber triggers rollback for stale pending orders.
    \item Staff heartbeat expiry subscriber marks staff offline and triggers reassignment logic.
    \item Graceful shutdown ensures clean closure of HTTP, socket, Redis, and PostgreSQL resources.
\end{itemize}
